## 为什么需要节流和防抖？

在日常开发中，某些事件会高频触发（如 `scroll`、`resize`、`input`），如果每次触发都执行回调函数，会造成严重的性能问题。

**节流（Throttle）** 和 **防抖（Debounce）** 就是用来限制函数执行频率的两种策略。

---

## 防抖（Debounce）

> 事件触发后 n 秒才执行，如果 n 秒内再次触发，则重新计时。

**典型场景**：搜索框输入联想、窗口 resize 事件

### 基础实现

```js
/**
 * 防抖函数
 * @param {Function} fn - 需要防抖的函数
 * @param {number} delay - 延迟时间（毫秒）
 * @returns {Function} 防抖后的函数
 */
function debounce(fn, delay = 300) {
  let timer = null

  return function (...args) {
    // 每次触发清除之前的定时器
    if (timer) clearTimeout(timer)

    // 重新设定定时器
    timer = setTimeout(() => {
      fn.apply(this, args)
      timer = null
    }, delay)
  }
}
```

### 进阶版（支持立即执行 + 取消）

```js
function debounce(fn, delay = 300, immediate = false) {
  let timer = null

  const debounced = function (...args) {
    if (timer) clearTimeout(timer)

    if (immediate && !timer) {
      // 立即执行一次
      fn.apply(this, args)
    }

    timer = setTimeout(() => {
      if (!immediate) {
        fn.apply(this, args)
      }
      timer = null
    }, delay)
  }

  // 取消方法
  debounced.cancel = () => {
    clearTimeout(timer)
    timer = null
  }

  return debounced
}
```

## 节流（Throttle）

> 在 n 秒内只执行一次，即使在 n 秒内多次触发。

**典型场景**：滚动加载、按钮频繁点击

### 时间戳实现

```js
/**
 * 节流函数（时间戳版）
 * 特点：首次触发立即执行，结束后不再执行
 */
function throttle(fn, interval = 300) {
  let lastTime = 0

  return function (...args) {
    const now = Date.now()

    if (now - lastTime >= interval) {
      fn.apply(this, args)
      lastTime = now
    }
  }
}
```

### 定时器实现

```js
/**
 * 节流函数（定时器版）
 * 特点：首次触发延迟执行，结束后还会执行一次
 */
function throttle(fn, interval = 300) {
  let timer = null

  return function (...args) {
    if (!timer) {
      timer = setTimeout(() => {
        fn.apply(this, args)
        timer = null
      }, interval)
    }
  }
}
```

## 对比总结

| 特性 | 防抖 Debounce | 节流 Throttle |
|------|:---:|:---:|
| **执行时机** | 停止触发后 | 固定间隔 |
| **执行次数** | 最后一次 | 均匀执行 |
| **搜索联想** | ✅ 适用 | ❌ 不适用 |
| **滚动加载** | ❌ 不适用 | ✅ 适用 |
| **窗口 resize** | ✅ 适用 | ✅ 适用 |
| **按钮点击** | ❌ 不适用 | ✅ 适用 |

> 💡 **记忆技巧**：防抖像电梯关门（有人来就重新等），节流像地铁发车（固定间隔发一趟）。
