## 为什么性能很重要？

Google 研究表明：

- 页面加载时间从 **1 秒增加到 3 秒**，跳出率增加 **32%**
- 加载时间超过 **5 秒**，跳出率增加 **90%**
- **53%** 的移动端用户在加载超过 3 秒后放弃

性能不仅影响用户体验，还直接影响 **转化率** 和 **搜索排名**。

## 一、加载性能优化

### 1.1 资源压缩与打包

```js
// vite.config.js - 开启 gzip 压缩
import compression from 'vite-plugin-compression'

export default defineConfig({
  plugins: [
    compression({
      algorithm: 'gzip',
      threshold: 10240, // 大于 10KB 才压缩
    })
  ],
  build: {
    // 代码拆分策略
    rollupOptions: {
      output: {
        manualChunks: {
          'vue-vendor': ['vue', 'vue-router', 'pinia'],
          'utils': ['lodash-es', 'dayjs'],
        }
      }
    }
  }
})
```

### 1.2 图片优化

图片通常占页面流量的 **60%** 以上，优化策略：

| 策略 | 说明 | 效果 |
|------|------|------|
| 格式选择 | WebP/AVIF 替代 PNG/JPG | 减少 25-50% 体积 |
| 懒加载 | `loading="lazy"` | 减少首屏请求 |
| 响应式图片 | `srcset` + `sizes` | 按屏幕加载适当大小 |
| CDN | 图片托管到 CDN | 加速分发 |

```html
<!-- 响应式图片 + 懒加载 -->
<img
  src="photo-800.webp"
  srcset="photo-400.webp 400w, photo-800.webp 800w, photo-1200.webp 1200w"
  sizes="(max-width: 600px) 400px, (max-width: 1000px) 800px, 1200px"
  loading="lazy"
  alt="示例图片"
/>
```

### 1.3 预加载关键资源

```html
<!-- 预加载关键字体 -->
<link rel="preload" href="/fonts/inter.woff2" as="font" type="font/woff2" crossorigin>

<!-- 预连接第三方域名 -->
<link rel="preconnect" href="https://api.example.com">

<!-- 预取下一页资源 -->
<link rel="prefetch" href="/next-page.js">
```

## 二、渲染性能优化

### 2.1 避免强制同步布局

```js
// ❌ 错误：读写交替触发多次重排
elements.forEach(el => {
  const width = el.offsetWidth     // 读 → 触发重排
  el.style.width = width + 10 + 'px'  // 写
})

// ✅ 正确：批量读，批量写
const widths = elements.map(el => el.offsetWidth)  // 批量读
elements.forEach((el, i) => {
  el.style.width = widths[i] + 10 + 'px'           // 批量写
})
```

### 2.2 使用 CSS contain 属性

```css
/* 告诉浏览器该元素的布局/绘制不影响外部 */
.card {
  contain: layout paint;
}

/* 对于下拉列表等可能变化大小的容器 */
.dropdown-content {
  contain: strict;
  content-visibility: auto;
}
```

### 2.3 虚拟化长列表

当列表项超过 **1000** 条时，应考虑使用虚拟列表（Virtual List）技术：

```vue
<template>
  <!-- 只渲染可视区域的 20-30 个 DOM 节点 -->
  <div class="virtual-container" @scroll="onScroll">
    <div :style="{ height: totalHeight + 'px' }">
      <div :style="{ transform: `translateY(${offsetY}px)` }">
        <div v-for="item in visibleItems" :key="item.id">
          {{ item.text }}
        </div>
      </div>
    </div>
  </div>
</template>
```

## 三、Vue 特定优化

### 3.1 合理使用 computed 和 watch

```js
// ✅ 优先使用 computed（有缓存）
const fullName = computed(() => `${firstName.value} ${lastName.value}`)

// ❌ 避免用 watch 做 computed 能做的事
watch([firstName, lastName], () => {
  fullName.value = `${firstName.value} ${lastName.value}`  // 不好！
})
```

### 3.2 组件懒加载

```js
// 路由级懒加载
const routes = [
  {
    path: '/dashboard',
    component: () => import('./views/Dashboard.vue')
  }
]

// 组件级懒加载
const HeavyChart = defineAsyncComponent(() =>
  import('./components/HeavyChart.vue')
)
```

### 3.3 v-once 和 v-memo

```vue
<template>
  <!-- 只渲染一次，后续不更新 -->
  <footer v-once>
    <p>© 2026 Vue Lab. All rights reserved.</p>
  </footer>

  <!-- 仅当依赖变化时才更新 -->
  <div v-memo="[item.id, item.selected]">
    <HeavyComponent :data="item" />
  </div>
</template>
```

| **Bundle Analyzer** | 打包体积分析 | ⭐⭐⭐⭐ |

### Core Web Vitals 指标

```
LCP（Largest Contentful Paint）  ≤ 2.5s   最大内容绘制
FID（First Input Delay）         ≤ 100ms  首次输入延迟
CLS（Cumulative Layout Shift）   ≤ 0.1    累积布局偏移
```

## 总结

性能优化是一个**渐进式**的过程，不要试图一次优化到极致。建议按照以下优先级：

1. **先测量**：用 Lighthouse 获取基准分数
2. **优化加载**：压缩、拆包、懒加载
3. **优化渲染**：减少重排、虚拟化、CSS contain
4. **框架层优化**：computed、组件懒加载、v-memo
5. **持续监控**：部署 Web Vitals 监控

> 💡 **核心原则**：不要过早优化，但也不要忽视性能。先让代码正确工作，再有针对性地优化瓶颈。
